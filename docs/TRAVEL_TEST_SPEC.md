# LocalHub 여행 취향 테스트 명세서

> 버전: 1.0  
> 확정일: 2026-07-14  
> 관련 API: `GET /api/travel-test/questions`, `POST /api/travel-test`

## 1. 목적

사용자가 5개의 질문에 답하면 여행 취향 점수를 계산해 하나의 대표 유형을 결정하고, 해당 유형에 맞는 구미·경북권 여행지를 추천한다.

내부에서는 안정적인 계산과 API 연동을 위해 영문 유형 코드를 사용하고, 사용자 화면에는 별도의 한글 유형명만 노출한다.

## 2. 여행 유형

| 내부 코드 | 사용자 표시명 | 결과 설명 | 대표 키워드 |
|---|---|---|---|
| `HEALING` | 고요한 쉼표 수집가 | 바쁘게 움직이는 것보다 한 장소에서 천천히 머무는 시간을 좋아하는 여행자예요. 자연을 감상하거나 조용한 길을 걸으며 여행 속 여유를 발견해요. | 자연, 산책, 휴식, 조용함 |
| `EXPLORER` | 에너지 만렙 탐험가 | 익숙한 구경보다 직접 움직이고 경험하는 순간을 좋아하는 여행자예요. 새로운 활동과 체험을 만날수록 여행이 더욱 즐거워져요. | 레포츠, 체험, 야외, 활동 |
| `CULTURE` | 이야기를 좇는 시간여행자 | 장소의 겉모습뿐 아니라 그 안에 담긴 역사와 이야기를 발견하는 여행자예요. 문화시설과 유적지를 둘러보며 여행의 의미를 기록해요. | 문화, 역사, 전시, 사진 |
| `FOODIE` | 한입에 담는 로컬 미식가 | 여행지의 분위기를 음식으로 기억하는 여행자예요. 유명한 식당부터 시장과 지역 축제까지, 그 지역만의 맛을 찾아다녀요. | 음식점, 시장, 축제, 특산물 |

### 화면 표시 원칙

- 사용자 화면에는 `HEALING`과 같은 내부 코드를 표시하지 않는다.
- 결과 화면에는 한글 유형명, 결과 설명, 추천 장소를 표시한다.
- 내부 코드와 점수 정보는 백엔드 계산과 응답 식별에만 사용한다.

## 3. 점수 계산 규칙

1. 사용자는 5개 질문에 모두 답해야 한다.
2. 각 질문에서는 하나의 선택지만 선택할 수 있다.
3. 각 선택지는 대표 유형에 2점을 부여한다.
4. 실제 성향이 겹치는 일부 선택지는 보조 유형에 1점을 부여한다.
5. 모든 답변의 유형별 점수를 합산한다.
6. 가장 높은 점수를 획득한 유형 하나를 최종 결과로 선정한다.
7. 동점이면 7장의 동점 처리 규칙을 적용한다.

## 4. 질문과 선택지

### 질문 1. 여행을 떠났을 때 가장 기대되는 순간은?

여행에서 가장 중요하게 생각하는 취향을 확인하는 핵심 질문이다. 동점 처리의 첫 번째 기준으로 사용하므로 보조 점수를 부여하지 않는다.

| 선택지 ID | 사용자 선택지 | 대표 유형 | 점수 |
|---|---|---|---|
| `Q1_A` | 자연 속에서 아무 생각 없이 여유롭게 쉬는 순간 | HEALING | HEALING 2 |
| `Q1_B` | 평소 해보지 못한 활동에 직접 도전하는 순간 | EXPLORER | EXPLORER 2 |
| `Q1_C` | 오래된 장소에 담긴 새로운 이야기를 발견하는 순간 | CULTURE | CULTURE 2 |
| `Q1_D` | 그 지역에서만 맛볼 수 있는 음식을 만나는 순간 | FOODIE | FOODIE 2 |

### 질문 2. 여행지에서 갑자기 3시간의 자유 시간이 생겼다면?

| 선택지 ID | 사용자 선택지 | 대표 유형 | 점수 |
|---|---|---|---|
| `Q2_A` | 가까운 공원이나 조용한 길을 천천히 걸어본다 | HEALING | HEALING 2, CULTURE 1 |
| `Q2_B` | 근처에서 할 수 있는 체험이나 액티비티를 찾아본다 | EXPLORER | EXPLORER 2 |
| `Q2_C` | 박물관·전시관·역사적인 장소를 찾아간다 | CULTURE | CULTURE 2 |

각 유형을 분명하게 구분하는 질문이다. 동점 처리의 두 번째 기준으로 사용한다.

| `Q3_A` | 마음이 편안해졌던 장소와 그곳의 분위기 | HEALING | HEALING 2 |
| `Q3_B` | 직접 움직이며 경험했던 짜릿한 활동 | EXPLORER | EXPLORER 2 |
| `Q3_C` | 장소에서 알게 된 역사와 흥미로운 이야기 | CULTURE | CULTURE 2 |
| `Q3_D` | 다시 먹고 싶을 정도로 인상 깊었던 음식 | FOODIE | FOODIE 2 |

## 6. 계산 예시

사용자가 `Q1_A`, `Q2_A`, `Q3_C`, `Q4_A`, `Q5_C`를 선택한 경우:

```text
HEALING  = 2 + 2 + 0 + 2 + 0 = 6
EXPLORER = 0 + 0 + 0 + 0 + 0 = 0
CULTURE  = 0 + 1 + 2 + 0 + 2 = 5
FOODIE   = 0 + 0 + 0 + 1 + 0 = 1
```

최종 결과는 `HEALING`, 사용자 표시명은 **고요한 쉼표 수집가**이다.

## 7. 동점 처리 규칙

최고 점수가 두 개 이상이면 아래 순서대로 최종 유형을 결정한다.

1. 동점 유형 중 질문 1에서 선택한 대표 유형이 있으면 해당 유형을 선택한다.
2. 결정되지 않으면 질문 3에서 선택한 대표 유형을 확인한다.
3. 결정되지 않으면 질문 5에서 선택한 대표 유형을 확인한다.
4. 끝까지 결정되지 않으면 `HEALING → EXPLORER → CULTURE → FOODIE` 고정 순서를 적용한다.

고정 순서는 성향의 우열이 아니라 같은 입력에 항상 같은 결과를 반환하기 위한 기술적 안전장치이다.

### 의사코드

```python
max_score = max(scores.values())
tied_types = [code for code, score in scores.items() if score == max_score]

if len(tied_types) == 1:
    return tied_types[0]

for question_id in [1, 3, 5]:
    primary_type = get_selected_primary_type(answers, question_id)
    if primary_type in tied_types:
        return primary_type

for code in ["HEALING", "EXPLORER", "CULTURE", "FOODIE"]:
    if code in tied_types:
        return code
```

## 8. 질문 조회 및 결과 제출 원칙

### 질문 조회

```http
GET /api/travel-test/questions
```

- FE에는 `questionId`, 질문 문구, `optionId`, 선택지 문구만 전달한다.
- `scores`와 `primaryType`은 사용자 화면에 전달하지 않는다.

### 답변 제출

```http
POST /api/travel-test
```

```json
{
  "answers": [
    { "questionId": 1, "optionId": "Q1_A" },
    { "questionId": 2, "optionId": "Q2_C" },
    { "questionId": 3, "optionId": "Q3_A" },
    { "questionId": 4, "optionId": "Q4_B" },
    { "questionId": 5, "optionId": "Q5_D" }
  ]
}
```

- FE는 점수를 계산하지 않고 선택한 질문·선택지 ID만 전송한다.
- BE 서비스가 `travel_test_questions.json`을 읽어 입력 검증과 점수 계산을 수행한다.
- 질문 누락, 중복 questionId, 존재하지 않는 optionId는 요청 오류로 처리한다.

## 9. 장소 추천 연결 규칙

1. 설문 점수로 내부 여행 유형을 결정한다.
2. `travel_candidates.json`에서 `recommendedTypes`에 해당 유형이 포함된 장소만 추출한다.
3. `reviewRequired`가 `true`인 후보는 추천에서 제외한다.
4. 유형 키워드와 장소 태그의 교집합을 `matchScore`로 계산한다.
5. `matchScore` 내림차순, 동점이면 `contentId` 오름차순으로 정렬한다.
6. 최대 3개 장소를 반환한다.
7. 유형 판정과 추천 순위는 결정론적 코드로 계산하며 OpenAI에 맡기지 않는다.
